This database models a Food Truck Network — a system where truck owners register their vehicles and book spots at various city locations. The owners table stores personal data of each truck owner with unique phone and email contacts. The trucks table links each truck to its owner via a foreign key and enforces a unique license plate. The locations table describes available market spots across different cities with a daily rental fee. The bookings table connects trucks with locations and records the date and cost of each booking. All date columns include a CHECK constraint ensuring dates are 2026-01-01 or later. Numeric columns like daily_fee, total_fee, and capacity have CHECK constraints preventing negative values. DEFAULT values are used for booking status, active flags, and dates. The schema has 3 foreign key relationships forming a clear relational structure between all 4 tables.
